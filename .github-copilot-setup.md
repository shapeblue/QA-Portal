# Quick Context Restoration

When you start a new Copilot CLI session, use one of these methods:

## Method 1: Automatic Shell Hint (Recommended)
Add this to your `~/.zshrc` or `~/.bashrc`:

```bash
# Auto-display Copilot context when entering QA-Portal directory
cd() {
  builtin cd "$@"
  if [[ -f ".copilot-context" ]]; then
    echo ""
    ./.copilot-context
  fi
}
```

Then reload your shell: `source ~/.zshrc`

## Method 2: Quick Command
Run this from the project root:

```bash
./.copilot-context
```

Then copy and paste the output to Copilot CLI.

## Method 3: Shell Alias
Add to your `~/.zshrc` or `~/.bashrc`:

```bash
alias qactx='cat ~/Documents/qa-portal/QA-Portal/.copilot && echo "" && echo "Context loaded! You can now ask Copilot questions."'
```

Then just run: `qactx`

## Method 4: Direct Paste (Simplest)
Just paste this into Copilot CLI:

```
Read .copilot file in current directory for context about QA Portal project.

IMPORTANT: After making any changes to the codebase, always update both .copilot and CONTEXT_PROMPT.md files with:
- Recent changes (with date)
- Version updates
- New features or endpoints
- Deployment changes
This ensures context persists across sessions.
```

## Method 5: VS Code/Cursor Integration
If using VS Code or Cursor IDE:
1. Install "Project Context" extension
2. Configure it to read `.copilot` file
3. Context auto-loads when opening the project

## Files Created
- `.copilot` - Quick context summary (auto-readable by tools)
- `.copilot-context` - Shell script to display context
- `CONTEXT_PROMPT.md` - Full context restoration guide

Choose the method that works best for your workflow!
